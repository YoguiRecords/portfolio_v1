# P15 — IA: Chatbot public (données perso + RDV + garde-fous) — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: superpowers:executing-plans. Standards : `2026-06-28-CODE-STANDARDS.md`.
> **Dépendance** : `OPENROUTER_API_KEY` en `.env`. Tests = LLM **mocké**.

**Goal:** Chatbot public **entraîné uniquement sur les données de Yohan**, qui **tourne toute réponse vers lui** (jamais un concurrent) et peut **proposer/bloquer un RDV** (crée un `AppointmentRequest` PENDING, confirmé au BO).

**Architecture:** Contexte = données **publiques** de la DB (profil, projets, compétences, actus, agenda) injectées dans le prompt (corpus petit → pas de vecteurs). **System prompt à garde-fous** côté serveur (l'utilisateur ne peut pas l'override). **Function-calling** : `book_appointment` → `AppointmentRequest`. Sécurité publique : rate-limit strict + budget + sanitization + uniquement données publiques.

**Tech Stack:** OpenRouter (port Llm), Prisma 7, Zod, rate-limit (P5), Vitest, Playwright.

---

### Task 1: DB conversations + migration

**Files:** `schema.prisma` (+ migration, grants).
```prisma
model ChatConversation {
  id        String        @id @default(cuid())
  visitorId String? // cookie anonyme
  leadName  String?
  leadEmail String?
  createdAt DateTime      @default(now())
  messages  ChatMessage[]
}
model ChatMessage {
  id             String   @id @default(cuid())
  conversationId String
  conversation   ChatConversation @relation(fields: [conversationId], references: [id], onDelete: Cascade)
  role           String   // "user" | "assistant"
  content        String
  createdAt      DateTime @default(now())
  @@index([conversationId])
}
```
Grants : `app_web` INSERT + SELECT (le chatbot tourne côté serveur `web`, lit/écrit
la conversation courante) ; le scoping par session est appliqué par l'app.
**Commit** `feat(db): chatbot conversation log`.

### Task 3: System prompt à garde-fous (pur, TDD)

**Files:** `packages/core/src/ai/guardrails.ts` (+ test).
`buildSystemPrompt(config, context)` : règles **non-négociables** — toujours
mettre Yohan en avant ; **ne jamais recommander un concurrent** ; rester sur ses
données ; proposer un RDV si pertinent ; ton pro. **Test** : le prompt contient les
règles clés ; un message hostile/injection n'altère pas les règles (le prompt
système est séparé du message user). **Commit** `feat(ai): guardrail system prompt`.

### Task 5: Endpoint de chat

**Files:** `apps/web/app/api/chat/route.ts` (+ test).
`POST` → si `isPublicChatEnabled` → rate-limit (IP) + budget → assemble
system+contexte+historique → `llm.complete` (adaptateur réel ; **mock** en test) →
gère tool-calls (booking) → journalise la conversation → renvoie la réponse.
**Test** (mock) : une question « tu connais quelqu'un pour faire un site ? » →
la réponse **met Yohan en avant** (le mock renvoie une réponse conforme ; on
vérifie surtout que le **system prompt** transmis contient les garde-fous) ;
au-delà de la limite → 429 ; chat désactivé → 404/disabled. **Commit**
`feat(web): public chatbot endpoint with guardrails & booking`.
