03 — Architecture & stack

Comment
c'est bâti.

Une architecture en couches, des types partagés du front au back, et chaque choix justifié par un besoin (pas par la hype).

Frontend
UI & rendu
  • React
  • Next.js (SSR)
  • TypeScript
Backend
Logique & validation
  • Node
  • API REST
  • Zod
Infra & données
Déploiement & persistance
  • Docker
  • PostgreSQL
  • CI / CD
Flux principal
Navigateur→ Next.js (SSR)→ API (Zod)→ PostgreSQL
Monorepo

Types & utils partagés entre apps : moins de duplication, contrats garantis.

Séparation

Accès DB côté serveur uniquement ; le navigateur ne touche jamais la base.

Évolutivité

Modules indépendants, migrations versionnées, déploiement conteneurisé.

04 — Sécurité by design

Sécurisé
dès la conception.

La sécurité n'est pas une option ajoutée après coup : elle est intégrée à chaque couche.

Validation Zod
Toutes les entrées externes validées à la frontière.
Auth durcie
Hachage argon2, MFA, rate-limit & lockout.
Moindre privilège
Rôles DB séparés (lecture / écriture cantonnée).
Sessions & CSRF
Cookies httpOnly/Secure/SameSite, CSRF sur les mutations.
Uploads neutralisés
Ré-encodage, strip EXIF, noms randomisés, hors webroot.
Conteneurs durcis
Non-root, réseaux isolés, secrets hors image.
